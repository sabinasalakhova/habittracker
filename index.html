Deploy Your Wellness Tracker with GitHub Pages

A step-by-step guide to hosting your wellness tracker for free using GitHub Pages

1Create a GitHub Account

If you don't have one already, go to github.com and sign up for a free account.

2Create a New Repository

Click the "+" icon in the top right and select "New repository".

Name your repository (e.g., "wellness-tracker").

Make it public (required for free GitHub Pages).

Initialize with a README (optional but recommended).

3Upload Your HTML File

In your new repository, click "Add file" â†’ "Upload files".

Drag and drop your HTML file or choose it from your computer.

Name it index.html for it to work as the main page.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Wellness Tracker</title> <!-- Your wellness tracker code here --> </head> <body> <!-- Your wellness tracker content here --> </body> </html>

4Enable GitHub Pages

Go to your repository's Settings tab.

Scroll down to the "GitHub Pages" section.

Under "Source", select the "main" branch (or "master" if that's what you have).

Click "Save".

5Access Your Live Website

Wait a minute or two for GitHub to deploy your site.

Refresh the GitHub Pages section in Settings to see your live URL.

It will follow this pattern: https://[your-username].github.io/[repository-name]

Congratulations! Your wellness tracker is now live on the internet!

6Custom Domain (Optional)

If you have a custom domain, you can add it in the GitHub Pages settings.

You'll also need to configure your DNS settings with your domain registrar.

FAQ

How long does it take for my site to go live?

Usually less than a minute, but it can take up to 10 minutes after you enable GitHub Pages.

Can I use a custom domain?

Yes, GitHub Pages supports custom domains. You'll need to configure your DNS settings.

Is there a limit to how much I can host?

GitHub Pages sites have a soft bandwidth limit of 100GB per month and a soft limit of 10 builds per hour.

Can I use JavaScript and CSS?

Yes, GitHub Pages supports static sites with HTML, CSS, and JavaScript.
